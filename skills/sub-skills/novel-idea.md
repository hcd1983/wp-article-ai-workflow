# Sub-skill: novel-idea

**所屬類型：** `novel`（小說）
**觸發時機：** `series:init --type novel` 完成後
- `source_mode = "ai"`：直接展開
- `source_mode = "interview"`：`skills/interview.md` 訪談結束，整理故事種子後觸發

---

## 輸入

從 `_<slug>-series-config.md` 讀取：

| 欄位 | 用途 |
|------|------|
| `theme` | 核心主題（例：「人類自由 vs AI 控制」） |
| `genre` | 題材（例：「科幻 / 反烏托邦」） |
| `style` | 文風（例：「dark dystopian」） |

若為 `interview` 模式，還需讀入訪談整理的故事種子：
- 核心主題（一句話）
- 主要角色雛型（名字 + 核心衝突）
- 故事開場場景
- 使用者特別強調的細節或情感

---

## Agent 執行步驟

1. **故事前提（Premise）**
   - 用 2–3 句話寫出故事的核心張力：發生了什麼事？主角面對什麼困境？讀者會想知道什麼？

2. **世界初稿**
   - 時代與場景設定（時間、地點、科技水準 / 社會結構）。
   - 世界的核心規則或限制（讓故事能發生的關鍵設定）。
   - 此世界的主要矛盾或衝突源頭。

3. **角色雛型（2–3 位）**
   - 每位角色：名字、角色定位（主角 / 對立者 / 輔助者）、一句話背景、驅動動機、與主題的關係。

4. **核心衝突**
   - 外部衝突：主角需要達成什麼、阻礙是什麼。
   - 內部衝突：主角內心的矛盾或成長弧線。

5. **情感核心**
   - 讀者讀完整個系列後，應該感受到什麼？（例：「即使在最黑暗的系統中，個體的抵抗仍有意義」）

---

## 輸出格式

完成後寫入 `_<slug>-series-config.md` 的以下欄位：

```json
{
  "world": "世界觀初稿：時代、場景、核心規則、主要矛盾的完整描述",
  "characters": [
    {
      "name": "角色名",
      "role": "主角 | 對立者 | 輔助者",
      "background": "一句話背景",
      "motivation": "驅動動機",
      "traits": ["特質1", "特質2"],
      "arc": "成長弧線描述"
    }
  ],
  "metadata": {
    "novel_premise": "故事前提：2–3 句話的核心張力描述",
    "core_conflict": {
      "external": "外部衝突描述",
      "internal": "內部衝突描述"
    },
    "emotional_core": "讀者應感受到的情感核心"
  }
}
```

## 潤稿檢查點

輸出後詢問：
> 「novel-idea 完成，要出動 editing assistant 潤稿嗎？」

- 是 → 潤稿 `world` 描述、`novel_premise`、`emotional_core` 的文字，覆寫後列出修改摘要。
- 否 → 直接進入下一子技能。
