# Sub-skill: story-angle

**所屬類型：** `investigative`（深度報導）
**觸發時機：** `series:init --type investigative` 完成後
- `source_mode = "ai"`：直接展開
- `source_mode = "interview"`：`skills/interview.md` 訪談結束（可用於與消息來源或主編的訪談筆記），整理報導種子後觸發

---

## 輸入

從 `_<slug>-series-config.md` 讀取：

| 欄位 | 用途 |
|------|------|
| `theme` | 報導主題（例：「台灣中小企業的 AI 導入困境」） |
| `genre` | 報導領域（例：「科技 / 產業」） |
| `style` | 報導語調（例：「深度分析、帶入真實案例」） |

若為 `interview` 模式，還需讀入訪談整理的報導種子：
- 初步線索或觀察
- 消息來源提供的資訊
- 記者 / 使用者的直覺與假設

---

## Agent 執行步驟

1. **核心提問**
   - 這個系列試圖回答什麼問題？用一句話寫出「核心問句」。
   - 例：「為什麼台灣中小企業明知 AI 能提升效率，卻普遍抗拒導入？」

2. **核心假設**
   - 目前對這個問題最合理的假設是什麼？
   - 例：「問題不在技術門檻，而在組織文化與決策流程的阻力。」
   - 說明這個假設可以如何被印證或推翻。

3. **新聞性與時效性**
   - 為什麼現在是報導這個議題的時機？（政策變動、市場事件、社會趨勢⋯⋯）
   - 這個報導對誰最重要？讀者讀完後能做什麼不同的決定？

4. **敘事線索（2–4 條）**
   - 可以從哪幾個角度切入這個主題？
   - 每條線索：角度名稱 + 一句話說明 + 這條線索能回答的子問題。
   - 例：
     - 線索一：「成本迷思」— 中小企業以為 AI 很貴，實際如何？
     - 線索二：「人才斷層」— 誰來做 AI 導入？內部缺什麼角色？
     - 線索三：「成功案例」— 哪些企業做到了？關鍵差異是什麼？

5. **利害關係人地圖**
   - 這個議題涉及哪些關鍵群體？
   - 每個群體：名稱、在議題中的角色、可能立場、是否為消息來源候選。

6. **系列架構初稿**
   - 根據敘事線索，提出各集的主題分工。
   - 這是 `investigation-structure` 子技能的前置輸入。
