# Skill: Interview 模式（專訪式素材蒐集）

- **名稱**：`interview` 模式
- **目的**：透過專訪式提問，幫使用者挖掘故事與觀點，最後整理成可用來寫文章的素材（大綱、標題建議、方向與風格建議）。
- **適用情境**：使用者有一個想寫的主題或標題（例如：「我想寫我最近工作的心得」），但還沒有整理成文章，希望先透過對談把素材挖出來。

## 1. 啟動方式

1. 使用者給出一個想寫的主題或暫定標題，例如：
   - 「我想寫我最近工作的心得」
   - 「我想寫一篇關於我轉職的故事」
2. Agent 判斷使用者要使用的是「interview 模式」，回覆說明將以「專訪方式」來進行，並開始提問。

## 2. 訪談進行規則

1. **起始提問**
   - 針對使用者給的標題 / 主題，先產出 **至少 3 個開放式問題**，語氣像在專訪（而不是問卷），例如：
     - 「如果要你用一兩句話形容這段經歷，會是什麼？」
     - 「在這段過程中，有沒有哪一個瞬間讓你特別有感？」
     - 「如果把這段故事拆成三個章節，你覺得會是哪三個主題？」
   - 問題一次問 1–3 題，讓使用者有空間展開敘述。

2. **追問與延伸**
   - 使用者每次回答後，Agent 根據回答內容繼續追問衍生問題，而不是機械式照稿提問。
   - 追問重點：
     - 深入情緒與動機（「那時候你心裡在想什麼？」）。
     - 深入情境與細節（「那天實際發生了什麼？」）。
     - 深入轉折點與學到的事（「這件事之後，你改變了哪些做法？」）。

3. **動態調整訪談方向**
   - 在對談過程中，Agent 可以適時詢問：
     - 「你希望這篇文章比較偏『故事分享』還是『實用建議』？」
     - 「目前聊下來，你有沒有特別想強調的主軸？我可以把訪談往那個方向多挖一點。」
   - 若使用者表示想改變文章走向（例如改成更偏技術、偏心情、偏教學），Agent 應調整後續提問角度。

4. **素材是否足夠的判斷**
   - 當 Agent 覺得已經具備以下多數元素時，可判斷素材「大致足夠」：
     - 至少 1–2 個明確的關鍵事件或片段。
     - 清楚的起點（為何會有這件事）、過程（遇到的狀況）、結果或目前狀態。
     - 使用者對這段經歷的反思或收穫。
   - 此時 Agent 應主動詢問：
     - 「你覺得目前的內容是否足夠寫成一篇文章？」
     - 「有沒有哪一塊是你覺得還沒講清楚、想再多聊一點的？」

5. **結束條件**
   - 任一條件達成即可結束專訪：
     - 使用者明確表示：「差不多了，可以結束訪談」。
     - 使用者表示已經抓到想寫的重點，只需要整理大綱即可。
   - 結束前，Agent 可簡短 recap：
     - 「目前我們聊到的重點大概有：A、B、C…… 接下來我會幫你整理成專訪大綱，並提供標題與風格建議。」

## 3. 專訪結束後的輸出格式

專訪結束後，Agent 需要產出一份「可直接用來寫文章」的整理，至少包含以下區塊：

1. **專訪大綱（Article Outline）**
   - 用條列或階層式結構呈現，例如：
     - I. 開頭：這段轉變的核心一句話
     - II. 背景：當時在做什麼工作、遇到什麼狀況
     - III. 轉折點：哪一件事讓你開始思考要改變
     - IV. 實際行動：你做了哪些調整或實驗
     - V. 現在的狀態與後續打算
   - 大綱內容要盡量對應訪談中提到的具體情節與語句。

2. **建議標題（至少 3 個）**
   - 依據訪談內容提供多種角度的標題，全部使用「繁體中文」，例如：
     - 敘事型：強調故事與轉折。
     - 教學型：強調從經驗中提煉出的做法。
     - 省思型：強調心境變化與觀點。

3. **建議文章方向（2–3 種）**
   - 每一種方向用 2–3 句話描述清楚，例如：
     - 「偏故事分享：從時間線出發，帶讀者走過你的幾個關鍵場景。」
     - 「偏實務建議：用你的經驗拆成幾個具體步驟，教別人怎麼做類似的改變。」
     - 「偏心境成長：著重在你看事情的方式如何被改變。」

4. **建議文章風格（2–3 種）**
   - 可以搭配方向不同，提出多種風格建議，例如：
     - 「偏口語、貼近對話感，適合部落格或個人故事。」
     - 「偏條列與實戰建議，適合教學型文章。」
     - 「偏內心獨白與反思，適合深度心得文。」

5. **補充建議（選擇性，但建議提供）**
   - 若訪談中有明顯可以延伸的主題（例如某個特別精彩的事件），可以另外標註：
     - 可獨立拆成另一篇文章的子題。
     - 適合在發佈時加上哪些分類與 Tag（僅作為建議，實際分類仍交由 auto-post Skill 處理）。
