# 文章插圖功能 — 實作計劃書

本文件描述「文章插圖」功能的需求與高階設計。**實作狀態**：已實作。腳本 `yarn ai:add-illustrations`、設定見 `.env` 與 `docs/illustration-rules.md`，流程見 `SKILL.md` §3 與步驟 3.5。

---

## 1. 目標與需求摘要

- **目標**：在不改動文章原文的前提下，依語意與使用者偏好，在文章中穿插適合的插圖。
- **行為**：Agent 依段落語意判斷插圖位置與風格；使用者可關閉插圖、設定上限與規則，也可在產出前或 interview 中指定位置與風格。

---

## 2. 功能需求

### 2.1 自動插圖

- Agent 依文章語意、段落內容判斷**適合的插圖位置**，穿插於文中。
- 不修改原文文字，只在適當位置插入圖片（markdown / HTML）。
- 每張插圖需有對應的 prompt 與 alt text／圖說。

### 2.2 每篇文章可關閉插圖

- 使用者可在單篇文章上**明確標記「這篇不要自動插圖」**（例如 frontmatter `illustration: off`）。
- 有此標記時，該篇文章不進行任何插圖生成與插入。

### 2.3 插圖數量與規則設定

- **數量上限**：可設定每篇文章最多幾張插圖（例如 `.env` 的 `ILLUSTRATION_MAX_PER_ARTICLE=3`）。
- **規則檔**：可用 `.md` 檔描述哪些**類型／標籤／分類**的文章不插圖（或較低上限），`.env` 放數值與開關。

### 2.4 風格與時機

- **預設**：插圖出現時機與風格由 Agent **自由判斷**。
- **使用者可指定**：
  - 在產出前或 **interview 對話中**指定：
    - 在「某段／某處」追加插圖；
    - 該插圖或整篇的**風格**（手繪、卡通、寫實、線條等）。
  - 使用者明確指定的位置與風格**優先**於 Agent 自由發揮。

### 2.5 風格一致性（同一篇文章）

- **原則**：同一篇文章內的所有插圖**風格必須一致**，以維持視覺與閱讀體驗一致。
- **例外**：若使用者在產出前或 interview 中**明確要求**某張（或某幾張）插圖使用不同風格，則以使用者指定為準，其餘插圖仍維持該篇文章的「主風格」。
- 實作時：先決定該篇的「主風格」（來自使用者整篇風格指示，或 Agent 依內容判斷），生成每張圖的 prompt 時都帶入此主風格；僅在使用者對特定位置有風格指示時，該張圖才覆寫風格。

---

## 3. 設定與規則

### 3.1 環境變數（.env）建議

| 變數 | 說明 | 範例 |
|------|------|------|
| `ILLUSTRATION_ENABLED_DEFAULT` | 預設是否啟用自動插圖 | `true` |
| `ILLUSTRATION_MAX_PER_ARTICLE` | 每篇最大插圖數 | `3` |
| `ILLUSTRATION_DEFAULT_STYLE` | 預設圖像風格（供 prompt 使用） | `clean-diagram` |

### 3.2 規則說明檔（.md）

- 用途：人類可讀的規範與範例（哪些類型不插圖、如何標記「本篇不插圖」等）。
- 可含 YAML 或清單格式，供程式解析「不插圖類型／標籤」等規則。

---

## 4. 技術流程要點

1. **載入設定**：讀取 `.env` 與規則檔，決定是否對該篇文章啟用插圖、上限與排除類型。
2. **本篇是否插圖**：檢查文章 frontmatter／metadata 與規則（類型／標籤），決定是否執行插圖流程。
3. **語意分析與候選位置**：分段、抽取主題與段落角色，產出候選插圖位置與建議圖型。
4. **套用上限與使用者指定**：鎖定使用者指定的位置與風格，其餘名額由 Agent 從候選中選取；**整篇確定一張「主風格」**（使用者整篇指定 或 Agent 判斷），並在生成每張圖的 prompt 時套用，僅對「使用者明確指定風格」的那幾張覆寫。
5. **生成 prompt 與 alt text**：對每個插圖位產生 image prompt 與無障礙說明。
6. **輸出**：插入圖片標記（placeholder 或實際路徑）到文章結構中。

---

## 5. 互動模式

- **單次指令**：使用者提供草稿 → Agent 分析並產出插圖計畫（位置＋風格一致）→ 確認後生成圖片並寫回。
- **Interview 模式**：對話中可隨時要求「某段後加一張圖、風格 Y」；Agent 記錄後，在該篇仍維持**其餘插圖風格一致**，僅指定處使用 Y。

---

## 6. 邊界與錯誤處理

- 文章過短或語意分散：可不插圖或僅建議封面圖，並提示使用者。
- 未設定上限：使用安全預設（如 2–3 張）。
- 使用者指定數超過全域上限：以全域上限為準並提示。
- 規則檔格式錯誤：採預設行為並 log 警告，不中斷主流程。
